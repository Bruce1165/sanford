# 周线老鸭头 - 三套进场信号完整设计

## 总体设计思路

老鸭头形态是一个完整的多阶段趋势，应该拆分为**三套独立的选股公式**，每套对应不同的进场时机和风险偏好。

---

## 周线老鸭头完整形态流程

```
颈（头）形成 → 调整缩量回调 → MA5/MA10金叉（鸭鼻孔）→ MA30 支撑 → 放量突破（鸭嘴开口）
```

---

## 一、信号一：鸭鼻孔（激进买点）

### 核心逻辑

```
5/10/30 周均线多头排列
局部高点识别（近 30 周最高点）
MA5/MA10 快速金叉 + 小空隙（空隙越小越强）
MA30 支撑确认（MA10 > MA30 或 MA10 > MA30）
缩量确认（回调期间成交量 < 80% 平均）
```

### 筛选条件

```
核心筛选：鸭鼻孔形态识别 ✓
流通市值：由 StockFilter 自动过滤 ✓
数据时长：≥ 60 周（可配置）
振幅限制：由用户决定是否添加
```

### 实现步骤

1. **数据获取**
   - 从数据库获取日线数据（days=300）
   - 转换为周线 OHLCV（按周聚合）
   
2. **EMA 计算**
   - MA5 = 5 周指数移动平均
   - MA10 = 10 周指数移动平均
   - MA30 = 30 周指数移动平均
   
3. **多头排列检测**
   - 检查：是否 MA5 > MA10 > MA30
   - 通过条件：至少有一个时期满足
   
4. **局部高点识别**
   - 在近 30 周内寻找最高点
   - 记录：最高价格和对应索引
   
5. **回调检测**
   - 从局部高点后，寻找第一个价格下跌点
   - 记录：回调起始索引
   
6. **缩量检测**
   - 计算回调期间的成交量平均值
   - 识别：成交量 < 平均值 80% 的周期为"缩量期"
   - 收集：所有缩量周期的起止索引
   
7. **金叉检测**
   - 在缩量期中寻找 MA5 穿越 MA10 的点
   - 计算：空隙大小 = (MA5 - MA10) / MA10 × 100%
   - 记录：金叉索引
   
8. **支撑确认**
   - 检查：金叉点的 MA30 值 > MA10（或 MA10 > MA30）
   - 通过条件：MA30 提供支撑
   
9. **置信度计算**
   - 空隙得分：1.0 - (空隙 / 5.0)，越小越强
   - 量能得分：min(成交量 / 回调期平均, 2.0)，越大越强
   - 支撑得分：min((MA30 - MA10) / MA10, 10.0)，越大越强
   - 综合得分：空隙 40% + 量能 30% + 支撑 30%
   - 范围：0.0 - 1.0（1.0 = 最高置信度）

## 二、信号二：鸭嘴（稳健主买点）

### 核心逻辑

```
5/10/30 周均线多头排列
MA5/MA10 二次金叉（第一次金叉后回落）
张口形态识别（5/10 周均线呈鸭嘴开口）
30 周线持续向上（中期多头趋势确认）
```

### 筛选条件

```
核心筛选：鸭嘴形态识别 ✓
首次金叉后回落：识别第一次 MA5/MA10 金叉点
二次金叉检测：在回调后识别第二次 MA5/MA10 金叉
张口确认：MA5 > MA10 且 MA5 > MA30（鸭嘴开口）
趋势确认：MA30 持续向上（至少最近 8 周）
```

### 实现步骤（基于信号一）

1. **首次金叉点识别**
   - 在完整数据中寻找第一次 MA5 穿越 MA10 的点
   - 记录：首次金叉索引和价格
   
2. **回调识别**
   - 从首次金叉点后，寻找价格下跌 > 3% 的点
   - 记录：回调低点
   
3. **二次金叉检测**
   - 在回调后，寻找 MA5 再次穿越 MA10 的点
   - 计算：与首次金叉的时间间隔
   - 通过条件：间隔 3-15 周为宜
   
4. **张口形态识别**
   - 检查 MA5 是否形成"鸭嘴"形态（MA5 > MA10 > MA30）
   - 计算：张口大小 = (MA5 - MA30) / MA10
   - 通过条件：张口大小 > 3%
   
5. **趋势确认**
   - 检查 MA30 在最近 8 周是否持续上升
   - 计算：MA30 斜率（线性回归）
   - 通过条件：斜率 > 0

## 三、信号三：突破鸭头顶前高（加速追买）

### 核心逻辑

```
5/10/30 周均线多头排列
局部高点识别（近 30 周最高点）
放量突破识别（突破局部高点时成交量放大）
右侧主升确认（鸭头右侧价格持续高于 MA30）
```

### 筛选条件

```
核心筛选：鸭头形态识别 ✓
左侧高点：识别近 30 周内最高点
右侧主升：鸭头后价格持续高于 MA30 至少 12 周
突破放量：突破当周量 > 前 4 周平均 2 倍
趋势确认：MA30 持续向上
```

### 实现步骤

1. **局部高点识别**
   - 同信号一
   
2. **放量突破识别**
   - 突破时成交量检测：突破周量 > 前 4 周均量 × 2
   - 记录：突破点索引和价格
   
3. **右侧主升确认**
   - 检查：突破后 12 周内 MA30 是否持续上升
   - 计算：MA30 斜率（线性回归）
   - 通过条件：斜率 > 0

## 实现架构设计

### 文件结构

```
screeners/
├── base_screener.py（基础类，共享）
├── signal_1_duck_nose.py（鸭鼻孔信号）
│   ├── class: DuckNoseScreener
│   ├── screen_stock()方法
│   └── get_config()配置
├── signal_2_duck_beak.py（鸭嘴信号）
│   ├── class: DuckBeakScreener
│   ├── screen_stock()方法
│   └── get_config()配置
└── signal_3_breakout.py（突破信号）
    ├── class: BreakoutScreener
    ├── screen_stock()方法
    └── get_config()配置
```

### 配置管理

每个 screener 有独立的配置：
```python
DEFAULT_CONFIG = {
    'duck_nose': {
        'min_weeks': 60,
        'ma5_period': 5,
        'ma10_period': 10,
        'ma30_period': 30,
        'local_high_window': 30,
        'volume_contraction_threshold': 0.8,
        'min_gap': 5.0,
    },
    'duck_beak': {
        'min_weeks': 60,
        'ma5_period': 5,
        'ma10_period': 10,
        'ma30_period': 30,
        'min_pullback_pct': 0.03,  # 回调至少 3%
        'max_pullback_interval_weeks': 15,  # 两次金叉间隔最多 15 周
        'min_beak_opening': 0.03,  # 张口至少 3%
        'trend_upweeks': 8,  # MA30 持续向上至少 8 周
    },
    'breakout': {
        'min_weeks': 60,
        'ma5_period': 5,
        'ma10_period': 10,
        'ma30_period': 30,
        'local_high_window': 30,
        'volume_breakout_threshold': 2.0,  # 突破量 > 2 倍
        'min_trend_upweeks': 12,  # 右侧主升至少 12 周
    }
}

# 从配置读取参数
def get_screener_config(screener_name):
    return DEFAULT_CONFIG.get(screener_name, {})
```
