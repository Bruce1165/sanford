# 需求处理工作流程

## 流程概览

```
业务团队提交需求
    ↓
需求收集与初步整理
    ↓
技术评审（你 + AI）
    ↓
需求确认（与业务团队）
    ↓
技术设计（AI 提供）
    ↓
设计方案确认
    ↓
分阶段开发实现
    ↓
测试与验证
    ↓
上线与反馈
```

---

## 详细步骤

### 第1步：业务团队提交需求

**谁做：** 业务团队成员

**怎么做：**
1. 复制 `docs/需求提交模板.md`
2. 填写完整的需求描述
3. 保存为 `docs/需求管理/待评审/REQ_XXX_功能名称.md`
4. 通知你（或提交到版本控制）

**关键点：**
- ✅ 使用模板，确保信息完整
- ✅ 清晰描述判断条件和规则
- ✅ 提供具体的示例场景
- ✅ 明确验收标准

---

### 第2步：需求收集与初步整理

**谁做：** 你

**做什么：**
1. 检查新提交的需求
2. 更新 `需求追踪表.md`
3. 标记状态为"待评审"

**输出：**
- 新需求已记录在追踪表中

---

### 第3步：技术评审

**谁做：** 你 + AI（我）

**做什么：**
1. 仔细阅读需求文档
2. 识别关键问题：
   - 需求是否清晰？
   - 是否缺少必要信息？
   - 技术可行性如何？
   - 是否依赖其他功能？
   - 估算开发工作量

3. 提出评审问题

**输出：**
- 评审意见清单
- 需要补充的信息

---

### 第4步：需求确认

**谁做：** 你 + 业务团队

**做什么：**
1. 与业务团队沟通评审问题
2. 补充缺失的信息
3. 调整需求描述（如果需要）
4. 业务团队确认最终需求

**输出：**
- 确认后的需求文档
- 移动到 `docs/需求管理/进行中/`

---

### 第5步：技术设计

**谁做：** AI（我）提供，你确认

**AI 提供：**
1. **数据模型设计**
   - 需要哪些表？
   - 字段定义
   - 索引设计
   - 关系图

2. **API 设计**
   - 需要哪些接口？
   - 请求/响应格式
   - 错误处理

3. **流程图**
   - 使用 Mermaid 绘制业务流程
   - 展示判断条件和分支逻辑

4. **架构设计**
   - 模块划分
   - 类设计
   - 配置文件结构

5. **实现计划**
   - 分阶段拆解
   - 每阶段的输出
   - 估时和里程碑

**输出：**
- `docs/需求管理/进行中/REQ_XXX_技术设计.md`

---

### 第6步：设计方案确认

**谁做：** 你 + 业务团队（可选）

**做什么：**
1. 你审查技术设计
2. 如果涉及界面，与业务团队确认
3. 提出修改意见（如果有）
4. 确认设计方案

**输出：**
- 确认后的技术设计文档

---

### 第7步：分阶段开发实现

**谁做：** AI（我）实现，你测试

**开发阶段：**
1. **阶段1：数据层**
   - 创建/修改数据库表
   - 编写数据访问代码
   - 迁移脚本

2. **阶段2：业务逻辑层**
   - 实现核心算法
   - 评分引擎
   - 规则引擎

3. **阶段3：API 层**
   - 实现接口
   - 参数验证
   - 错误处理

4. **阶段4：前端层**（如果需要）
   - UI 组件
   - 数据展示
   - 交互逻辑

5. **阶段5：配置层**
   - 配置文件
   - 参数可调化

**输出：**
- 每个阶段的代码
- 更新 `docs/需求管理/进行中/REQ_XXX_开发进度.md`

---

### 第8步：测试与验证

**谁做：** 你

**测试内容：**
1. 功能测试
   - 按照验收标准逐项测试
   - 边界条件测试
   - 异常情况测试

2. 数据测试
   - 验证数据准确性
   - 验证历史数据

3. 性能测试
   - 响应时间
   - 大数据量测试

**输出：**
- 测试报告
- Bug 清单（如果有）

---

### 第9步：上线与反馈

**谁做：** 你

**做什么：**
1. 部署到生产环境
2. 观察运行情况
3. 收集业务团队反馈
4. 记录问题

**输出：**
- 功能上线
- 反馈记录
- 后续优化建议（如果有）

**最终输出：**
- 移动到 `docs/需求管理/已完成/`
- 更新 `需求追踪表.md`

```
    └── REQ_000_示例需求.md
```

## 沟通建议

### 对业务团队：
1. **使用模板** - 确保信息完整
2. **描述判断条件** - 清晰的 if/else 逻辑
3. **提供示例** - 具体的场景最有效
4. **及时反馈** - 技术评审后尽快确认

### 对你（技术负责人）：
1. **及时响应** - 新需求尽快评审
2. **积极沟通** - 不清楚的地方主动问
3. **测试充分** - 上线前充分测试
4. **记录完整** - 保存所有设计文档和决策

### 对 AI（我）：
1. **提供技术设计** - 数据模型、API、流程图
2. **实现代码** - 按照设计方案编码
3. **解答疑问** - 技术问题随时解答
4. **调整方案** - 根据反馈灵活调整

## 快速启动指南

**业务团队首次提需求：**
1. 阅读并理解 `需求提交模板.md`
2. 参考下面的示例需求
3. 按模板填写需求
4. 提交到 `docs/需求管理/待评审/`

**你收到新需求后：**
1. 检查需求完整性
2. 如有问题，联系业务团队补充
3. 信息完整后，告诉我开始技术评审
4. 我会提供技术设计方案
5. 你确认后开始开发
