## 🚀 快速开始

### 对于业务团队成员

**第一次提需求，请按以下步骤：**

1. **阅读示例需求**
   - 查看 `已完成/REQ_000_示例_跨筛选器股票跟踪.md`
   - 了解如何填写需求文档

2. **使用模板填写需求**
   - 复制 `需求提交模板.md`
   - 按照示例填写完整的需求描述
   - 特别注意：
     - ✅ 清晰描述判断条件和规则
     - ✅ 提供具体的示例场景
     - ✅ 明确验收标准

3. **提交需求**
   - 保存为 `docs/需求管理/待评审/REQ_XXX_功能名称.md`
   - 通知技术负责人（你）
   - 等待技术评审

### 对于技术负责人（你）

**收到新需求后，请按以下步骤：**

1. **检查需求完整性**
   - 使用模板填写了吗？
   - 信息是否完整？
   - 判断条件是否清晰？

2. **如果有问题**
   - 联系业务团队补充信息
   - 等待补充完整后再继续

3. **如果信息完整**
   - 告诉 AI（我）："开始技术评审 REQ_XXX"
   - 我会提供技术设计方案

4. **评审技术设计**
   - 检查数据模型设计
   - 检查 API 设计
   - 检查流程图是否合理
   - 提出修改意见

5. **确认设计后**
   - 告诉 AI（我）："开始开发 REQ_XXX"
   - 我会分阶段实现功能

6. **测试验证**
   - 按验收标准逐项测试
   - 记录问题
   - 反馈给 AI 修复

7. **上线**
   - 部署到生产环境
   - 移动需求到 `已完成/`
   - 更新 `需求处理流程.md` 中的追踪表

## 💡 关键建议

### 对业务团队

1. **提供具体示例**
   - 不要只说"计算评分"
   - 要说"股票A在X日期，因为Y原因，评分应该是Z"

2. **明确判断条件**
   - 不要只说"根据情况调整"
   - 要说"如果成交量突破2倍，加0.2分"

3. **设定验收标准**
   - 不要只说"功能正常"
   - 要说"5000只股票评分计算时间<30秒"

4. **及时沟通**
   - 技术评审后尽快补充信息
   - 有疑问主动沟通

### 对技术负责人（你）

1. **及时响应**
   - 新需求尽快评审
   - 不清楚的地方主动问

2. **充分测试**
   - 上线前充分测试
   - 不要只做"表面测试"

3. **记录完整**
   - 保存所有设计文档
   - 记录重要的决策和原因

### 对 AI（我）

1. **提供完整设计**
   - 不要只给代码片段
   - 要给完整的方案和理由

2. **主动提问**
   - 不清楚的地方主动问
   - 提供建议和优化

3. **灵活调整**
   - 根据反馈修改方案
   - 不要固执己见

## 📞 常见问题

### Q: 业务团队不会写技术文档怎么办？
**A:** 没关系！使用模板，用文字描述业务逻辑即可，我会帮你转化为技术方案。

### Q: 判断条件很复杂，怎么描述？
**A:** 可以分步骤写，或者用 "如果...那么...否则..." 的句式。我也会画出流程图让你确认。

### Q: 需求可以修改吗？
**A:** 可以！在开发前的任何阶段都可以修改。开发开始后，小改没问题，大改需要评估。

### Q: 一个功能需要多长时间？
**A:** 取决于复杂度。我会先评估，然后给你分阶段的开发计划。

### Q: 可以并行开发多个功能吗？
**A:** 如果不冲突可以。但建议优先级高的先做，避免资源冲突。
