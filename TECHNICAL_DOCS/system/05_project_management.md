## 1. 目的

1. 记录每一次对五图链路或主干系统的改动，确保可追溯。
2. 记录关键决策理由，防止后续因人员变动或记忆模糊而重复踩坑。
3. 作为回滚或复盘时的第一手证据，降低事故定位成本。

## 2. 记录时机（必须）

1. 每次发布（无论成功或回滚）。
2. 每次参数或配置变更。
3. 每次出现 P0/P1 故障或回滚。
4. 每次主干代码合并或冻结区变更。
