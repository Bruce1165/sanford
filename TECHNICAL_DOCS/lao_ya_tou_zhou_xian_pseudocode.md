## 三、信号一：激进买点・鸭鼻孔缩量金叉（左侧低吸）

### 3.1 核心逻辑

**触发条件组合：**

```
条件A：MA5上穿MA10 (金叉)
  AND
条件B：MA5快速回到原位 (调整期 < 3周)
  AND
条件C：当前周量 >= pullback平均量 * 1.5 (放量确认)
  AND
条件D：当前周量 < 整体平均量 * 0.7 (非过度放量)
  AND
条件E：当前价格 < 30周均线 (未破颈)
  AND
条件F：鸭鼻孔闭合 (MA5回到原位下方)
  AND
条件G：阶段地量判断 (pullback期平均量 < 整体平均量 * 0.7)

→ 信号一触发
```

### 3.2 置信度计算公式

```
置信度 = (缺口大小 * 0.3) + (量比 * 0.4) + (支撑距离 * 0.3)

其中：
- 缺口大小 = MA10在金叉点的价格 - MA5在金叉点的价格
  标准化：缺口越大，得分越低 (0.1-0.5分)
- 量比 = 当前周量 / pullback平均量
  标准化：量比适中得分越高 (1.2-2.0分)
- 支撑距离 = (MA30在金叉点的价格 - MA5在金叉点的价格) / MA5价格
  标准化：支撑越近得分越高 (0.1-0.5分)
```

### 3.3 仓位控制规则

```
IF 缺口 <= 1.5% AND 量比 >= 1.8:
    仓位 = 4万手 (大仓位)
ELIF 缺口 <= 2.5% AND 量比 >= 1.5:
    仓位 = 3万手 (中仓位)
ELSE:
    仓位 = 2万手 (小仓位)
```

### 3.4 止损条件

```
止损价 = 30周均线当前价

触发：周线收盘价 < 止损价
无时间限制，无条件离场
```

### 3.5 适用条件

```
IF 流通市值 < 40亿 OR 流通市值 > 100亿:
    不触发信号一 (不适配左侧低吸)
```

## 四、信号二：核心主买・鸭嘴开口金叉（稳健最优）

### 4.1 核心逻辑

**阶段一：鸭嘴开口识别**

```
条件：MA5上穿MA10 (二次金叉)
```

**阶段二：确认三线多头排列持续**

```
FOR i = 金叉点 TO 金叉点+3:
    IF MA5[i] > MA10[i] > MA30[i] AND
       MA5[i-1] > MA10[i-1] > MA30[i-1]:
        多头排列持续时间 += 1
        IF 多头排列持续时间 >= 3:
            三线排列确认 = True
            BREAK

IF 三线排列确认 = True:
    进入阶段三（量能确认）
```

**阶段三：缩量后温和放量确认**

```
计算 pullback期平均量

FOR i = 金叉点 TO 金叉点+3:
    IF 周量[i] < pullback平均量 * 0.7:
        缩量期持续时间 += 1
    ELSE:
        缩量期持续时间 = 0

计算缩量期后2-4周量
IF 缩量期持续时间 >= 2 AND
   周量[金叉点+2 TO 金叉点+4].mean() >= pullback平均量 * 1.2 AND
   周量[金叉点+2 TO 金叉点+4].mean() < pullback平均量 * 2.0:
    缩量后放量确认 = True
```

### 4.2 置信度计算公式

```
置信度 = (缩量确认 * 0.4) + (三线排列 * 0.3) + (趋势向上 * 0.3)

其中：
- 缩量确认：缩量后放量确认得1分，否则0分
- 三线排列：持续周数越多得分越高 (3-6周得0.3-1.0分)
- 趋势向上：MA30斜率为正得1分，否则0分
```

### 4.3 仓位控制规则

```
IF 缩量后放量确认 AND 三线排列持续 >= 5周:
    仓位 = 6万手 (大仓位)
ELIF 缩量后放量确认 AND 三线排列持续 >= 3周:
    仓位 = 5万手 (中仓位)
ELSE:
    仓位 = 3万手 (小仓位)
```

### 4.4 止损条件

```
止损价 = 金叉当周K线的下沿价格

触发：周线收盘价 < 止损价
时间止损：4周未突破前高
```

### 4.5 适用条件

```
无限制（常态行情）
```

## 五、信号三：加速追买・放量突破鸭头前高（右侧主升）

### 5.1 历史高点识别

```
FOR 每只股票：
    维护历史局部高点列表 (最近100个)
    更新规则：高于前高2%则视为新高点
    存储时间、价格、周量
```

### 5.2 突破检测逻辑

```
条件A：当前价突破历史高点
条件B：突破周量 > 过去8周平均量 * 1.5 (大幅放量)
条件C：MA30当前价 > MA30[过去4周价] (趋势向上)
条件D：5/10/30周均线多头排列 (MA5>MA10>MA30)

全部满足 → 信号三触发
```

### 5.3 置信度计算公式

```
置信度 = (突破确认 * 0.4) + (放量确认 * 0.3) + (趋势确认 * 0.3)

其中：
- 突破确认：突破后3周仍在前高上方得1分
- 放量确认：突破后1周仍放量得1分，否则0分
- 趋势确认：MA30斜率为正得1分，否则0分
```

### 5.4 仓位控制规则

```
基础仓位 = 4万手

IF 突破确认 >= 3 AND 放量确认 = True:
    板块共振加成：
    检查目标板块是否同步突破
    若3个以上板块同步突破，仓位 = 5万手
    ELSE:
    仓位 = 4万手
```

### 5.5 止损条件

```
止损价 = 突破前高位置 (回调支撑位)

触发：周线收盘价 < 止损价
时间止损：3周未创更高点
```

### 5.6 适用条件

```
大盘上涨阶段 (板块共振)
```

## 六、主要检测流程

### 6.1 数据加载与预处理

```
步骤1：加载日K线数据
    - 从数据库加载近300个交易日
    - 转换为周K线 (聚合每周开盘/最高/最低/收盘/成交量)

步骤2：计算基础指标
    - 计算 MA5、MA10、MA30 (周线)
    - 识别局部高点 (窗口30周)
    - 计算整体平均量 (全历史)
    - 计算每个阶段平均量
```

### 6.2 并行信号检测

```
步骤3：并行检测三套信号
    - 启动三个检测器 (信号一、信号二、信号三)
    - 每个检测器独立运行
    - 收集所有检测结果

步骤4：信号合并
    - 按股票代码合并多套信号
    - 计算每套信号的置信度
    - 标记信号类型
```

### 6.3 信号筛选与排序

```
步骤5：筛选信号
    - 基于参数阈值筛选
    - 应用适用条件过滤
    - 过滤低置信度信号

步骤6：排序与选择
    - 对每套信号按置信度降序排序
    - 选择每套的最优信号
```

### 6.4 止损与仓位计算

```
步骤7：计算止损价
    - 基于每套信号的止损规则计算止损价
    - 附加到信号对象

步骤8：计算建议仓位
    - 基于信号类型和置信度计算建议仓位
    - 附加到信号对象
```

## 八、边界情况处理

### 8.1 数据不足

```
IF 周K线数据 < self.min_weeks * 5:
    RETURN None (数据不足)
```

### 8.2 异常值处理

```
IF MA值或成交量出现NaN或None:
    SKIP当前信号检测
    记录警告日志
```

### 8.3 并发信号处理

```
IF 同一只股票触发多套信号:
    记录所有信号
    按置信度排序，选择最优信号
    不丢弃其他信号（保留在结果中供参考）
```
