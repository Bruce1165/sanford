## 项目概述

**项目名称**: Neo Trading Analytics - 股票技术分析系统  
**目标用户**: Bruce (个人投资者)  
**核心功能**: A股数据获取、多维度筛选、可视化分析、结果导出  
**数据来源**: Baostock (A股历史数据)  
**技术栈**: Python + SQLite + Flask + React/ECharts

## 一、数据下载 (Data Acquisition)

### 1.1 功能需求

| 需求 | 优先级 | 状态 | 说明 |
|------|--------|------|------|
| 全量数据下载 | P0 | ✅ | 4663只股票完整数据 |
| 增量更新 | P0 | ✅ | 每日自动下载最新数据 |
| 断点续传 | P0 | ✅ | 中断后可恢复下载 |
| 数据去重 | P0 | ✅ | 防止重复数据插入 |
| 退市股处理 | P1 | ✅ | 识别并标记退市股票 |
| 数据校验 | P1 | ✅ | 检查数据完整性 |

### 1.2 数据范围

- **股票基础信息**: stocks表 (代码、名称、行业、板块等)
- **日行情数据**: daily_prices表 (开高低收、成交量、成交额、涨跌幅等)
- **公告数据**: announcements表 (公司公告)
- **涨停原因**: limit_up_reasons表 (涨停原因分析)

### 1.3 自动化要求

- **运行时间**: 每日收盘后自动运行 (建议15:30后)
- **运行方式**: 无需人工干预
- **失败处理**: 自动重试3次，失败则告警
- **完成通知**: 下载完成后自动运行筛选器

### 1.4 监控指标

- 数据完整性: 100% (4663只)
- 数据时效性: T+1 (次日可用)
- 下载成功率: >99%

## 五、个股筛选确认 (Stock Verification)

### 5.1 核心需求

用户可以在 Dashboard 中：
1. 查看某只股票是否符合特定筛选器条件
2. 查看该股票被哪些筛选器选中
3. 查看该股票的历史筛选记录

### 5.2 功能需求

| 需求 | 优先级 | 状态 | 说明 |
|------|--------|------|------|
| 单股筛选检查 | P1 | 📋 | 输入代码检查是否符合 |
| 筛选历史查询 | P2 | 📋 | 查看该股被选中历史 |
| 多筛选器对比 | P2 | 📋 | 同时检查多个筛选器 |
| 筛选条件展示 | P2 | 📋 | 显示具体匹配条件 |
| 不符合原因 | P2 | 📋 | 显示为何不符合 |

### 5.3 交互设计

```
用户操作:
1. 点击股票代码 → 弹出详情Modal
2. 选择"筛选检查"Tab
3. 系统显示:
   - 该股票被哪些筛选器选中
   - 选中日期
   - 具体匹配条件
   - 历史选中记录
```

### 5.4 技术实现

- **查询方式**: 根据股票代码反查筛选结果
- **数据源**: SQLite结果表
- **响应时间**: <500ms
