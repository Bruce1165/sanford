# NeoTrade2 发布与回滚手册（Flask + Cpolar）

> 版本：v1  
> 状态：生效中  
> 更新日期：2026-04-19

## 1. 目标

本手册用于解决并降低以下问题：

1. 发布后服务不可用
2. 认证口径不一致导致 401
3. 页面可开但关键接口不可用
4. 发布后回退不清晰，耗费大量调试时间

## 2. 发布范围原则

1. 仅在“验证与门禁手册”通过后发布
2. 发布过程必须记录版本标识与回滚点
3. 发布后先验证 API，再验证页面

## 3. 发布前门禁（全部通过才可发布）

### 3.1 代码与构建

1. Python 语法检查通过（后端与五图脚本）
2. 前端 build 通过
3. 关键配置文件存在且语法正常

### 3.2 服务与接口

1. `/api/health` 返回正常
2. 核心业务接口返回正常
3. 日志目录可写

### 3.3 数据与业务

1. 五图去重检查 SQL 通过（无重复）
2. CHECK 小样本回放通过
3. 参数快照版本可追溯

## 4. 标准发布流程（固定顺序）

1. 冻结改动并记录发布版本 ID
2. 执行门禁检查并保存结果
3. 重启 Flask 服务
4. 验证本地访问（`localhost`）
5. 验证外网访问（`cpolar` 域名）
6. 验证关键接口与最小业务闭环
7. 观察 15 分钟日志与错误率

## 5. 快速故障定位（3 分钟）

### 5.1 健康检查失败

优先排查：

1. Flask 进程是否正常
2. 环境变量是否加载（含认证配置）
3. 端口占用冲突

### 5.2 本地通、外网不通

优先排查：

1. Cpolar 隧道状态
2. 域名映射与证书状态
3. 本机网络出口与防火墙

### 5.3 页面可开但接口 401/403

优先排查：

1. 前端认证头与后端认证口径是否一致
2. Launchd 环境变量与运行时环境是否一致
3. 是否存在硬编码口令与环境口令冲突

### 5.4 接口 500

优先排查：

1. `logs` 中 stderr 最近错误
2. 数据库连接与锁状态
3. 参数解析与 JSON 序列化错误

## 6. 回滚预案（必须可执行）

### 6.1 触发条件

满足任一条件立即回滚：

1. 核心接口不可用
2. 结果出现重复写入
3. 状态推进错误
4. 关键业务闭环失败

### 6.2 回滚步骤

1. 切回上一稳定版本
2. 重启服务
3. 执行最小验收：健康检查 + 核心接口 + 小样本回放
4. 记录回滚原因与影响范围

### 6.3 回滚后要求

1. 未定位根因前禁止再次发布
2. 必须形成一次事故摘要与修复计划

## 7. 日常低成本巡检

每天执行：

1. `health` 检查一次
2. 五图去重 SQL 一次
3. 未处理池数量趋势检查一次
4. 错误日志增量检查一次
5. 当日变更记录一次
