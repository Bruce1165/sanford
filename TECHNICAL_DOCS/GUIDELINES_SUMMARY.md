### 🔧 Development Guidelines
**Primary File**:
- `DEVELOPMENT_BEST_PRACTICES.md` - Complete software engineering standards

**9 Critical Principles**:
1. ✅ Planning first (make development plan, write pseudocode)
2. ✅ Pseudocode before real coding
3. ✅ Update pseudocode before real code
4. ✅ Backup before modifying
5. ✅ English-only code (no Chinese characters)
6. ✅ Delete backup after verification
7. ✅ Debug with messages (print statements)
8. ✅ Stop when stuck (15 minutes circular problems)
9. ✅ Real data only (no placeholders)
10. ✅ Clean logs after each work chunk

**Purpose**: Ensure guidelines are ACTIVELY reviewed, not passively available

- [ ] Complete session start verification checklist
- [ ] Read ALL documentation guidelines (explicit acknowledgment)
- [ ] Read ALL development guidelines (explicit acknowledgment)
- [ ] Review project status and configuration
- [ ] Identify working directory and task scope

**Non-Compliant Response Prevention**:
- Rejects: "Okay", "I'll try", "Sure"
- Requires: Explicit "I have read and understand..." confirmations
- Examples: "I have read the documentation structure and I will follow the 4-level semantic hierarchy"

**Ready to Work Indicator**:
- Only complete when all checklist items acknowledged with explicit confirmations
- Claude Code must state: "I have read and understand [X, Y, Z...] and I commit to following these rules"
